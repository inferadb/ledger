## Local Golden Rules

1. **Never hand-edit `src/generated/`.** `PreToolUse` hook blocks it. Edit `proto/ledger/v1/*.proto` and run `just proto` to regenerate.
2. **Conversions stay slug-typed on both sides.** Proto messages carry `{Entity}Slug { slug: u64 }`; conversion functions operate on slugs. Slugâ†’ID translation lives in `crates/services/src/services/slug_resolver.rs` â€” never here. Importing `crates::state::SystemKeys` or `{Entity}Id` into conversion code is a layering violation.
3. **`TryFrom` for fallible (wireâ†’domain with validation); `From` for infallible (domainâ†’wire, trivial enum mappings).** A fallible `From` impl that panics on bad input is a bug â€” use `TryFrom`.
4. **Every new proto message gets at least one round-trip proptest in `convert/tests.rs`.** "It compiles" isn't coverage; proptest catches the invariants.
5. **`ErrorDetails` is the only structured-error wire format.** Don't invent per-service error message types. All service errors flow through `build_error_details()` (in `services/src/services/error_details.rs`).
6. **Breaking changes to released proto messages require a versioning plan.** Removing or renaming a field in `ledger.v1` is a break â€” add the new shape in `ledger.v2` or under a new request type.
