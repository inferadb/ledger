# proto

Protobuf codegen and `From`/`TryFrom` conversions between wire types and domain types. Cross-cutting — depended on by `services`, `sdk`, and sometimes `state`.

## Hard rules

- **Never edit `src/generated/`.** It is regenerated by `just proto` from `proto/ledger/v1/*.proto`. Hand-edits are silently overwritten and the PreToolUse hook blocks edits to this path.
- **Never hand-edit `Cargo.lock`.** The PreToolUse hook blocks it — update `Cargo.toml` and run `cargo +1.92 build --workspace` instead.
- All conversions live in `src/convert/**`, one file per domain (one per service or entity family).

## Dual-ID at the wire boundary

- External messages use `{Entity}Slug { slug: u64 }` — e.g., `VaultSlug`, `OrganizationSlug`.
- Do NOT translate slug → internal ID inside proto conversions. Conversion is `proto ↔ proto-domain`; the slug/ID translation happens in `services` via `SlugResolver`.
- Keep domain types on the proto side as slug-typed; the service layer injects `i64` IDs when it calls the state layer.

## `ErrorDetails`

- `ErrorDetails` proto (in `ledger.proto`) carries `error_code`, `is_retryable`, `retry_after_ms`, `context` map, `suggested_action`.
- Built by `raft::services::error_details::build_error_details` (single builder — no specialized helpers).
- Attached to gRPC `Status` by `raft::services::metadata::status_with_correlation` when `status.details().is_empty()`.
- SDK decodes it into `sdk::error::ServerErrorDetails` (boxed to keep `Result` small — `clippy::result_large_err`).

## Workflow

1. Edit `proto/ledger/v1/*.proto`.
2. Run `just proto` (regenerates `src/generated/ledger.v1.rs`).
3. Add or update conversion in `src/convert/{domain}.rs` — see `/add-proto-conversion` skill.
4. If the change adds a new RPC, follow `/new-rpc` skill.
5. `just ready` (proto + fmt + clippy + test) before declaring done.
