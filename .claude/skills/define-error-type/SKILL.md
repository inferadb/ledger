# define-error-type

Error handling in InferaDB is load-bearing for observability (correlation IDs, `ErrorDetails` on gRPC responses, structured log fields) and for classification (retryability, suggested actions). A sloppy variant breaks all of that.

## Crate boundary

| Crate | Error library |
| --- | --- |
| `types`, `store`, `proto`, `state`, `consensus`, `raft`, `services`, `server` | **snafu only** |
| `sdk` | **thiserror only** (consumer-facing types: `SdkError`, `ResolverError`) |

Never `anyhow`. Never mix snafu + thiserror inside one crate.

## Variant shapes

### Source-bearing variant ‚Äî REQUIRES implicit location

```rust
#[derive(Debug, snafu::Snafu)]
pub enum MyError {
    #[snafu(display("Storage error: {source}"))]
    Storage {
        source: inferadb_ledger_store::StoreError,
        #[snafu(implicit)]
        location: snafu::Location,
    },
}
```

The `#[snafu(implicit)] location: snafu::Location` field is **required** on every variant that carries a `source`. It captures the `file:line` where `.context(‚Ä¶)?` was called ‚Äî this is what makes snafu traces actionable.

### Leaf variant ‚Äî no location needed

```rust
#[snafu(display("Invalid key length: {len}"))]
InvalidKeyLength { len: usize },
```

Leaf variants (no `source`) do not need a `location` field.

### API-facing variant ‚Äî `message: String`

```rust
#[snafu(display("Rate limited: {message}"))]
RateLimited { message: String },
```

Anything that crosses a gRPC boundary serializes its message into `tonic::Status`. Use a plain `String` field named `message` ‚Äî structured context belongs in the `ErrorDetails` proto, not inside the error display.

## Propagation

```rust
some_operation().context(StorageSnafu)?;
```

- Always `.context(XxxSnafu)?`.
- **Never** construct variants manually: `MyError::Storage { source, location: ... }` bypasses the implicit-location capture and reports the wrong line.
- `StorageSnafu` is generated by the `Snafu` derive ‚Äî available without import inside the impl/module.

## `ErrorCode` integration

Every server-crate error variant must map to a variant of `ErrorCode` in `crates/types/src/error_code.rs`. Typical methods on the error type:

```rust
impl MyError {
    pub fn code(&self) -> ErrorCode { /* map variant ‚Üí ErrorCode */ }
    pub fn is_retryable(&self) -> bool { matches!(self.code(), ErrorCode::Unavailable | ...) }
    pub fn suggested_action(&self) -> Option<&'static str> { /* per-variant guidance */ }
}
```

`ErrorDetails` (in `raft::services::error_details::build_error_details`) picks up `code`, `is_retryable`, and `suggested_action`. Skipping these means the SDK loses classification ‚Äî `RateLimited` becomes `Unknown` to consumers.

## gRPC response

Let `raft::services::metadata::status_with_correlation()` build the `tonic::Status` ‚Äî it attaches `ErrorDetails` automatically when `status.details().is_empty()`. Do not construct `Status::new(Code::Internal, "...")` by hand in service handlers.

## SDK side

```rust
#[derive(thiserror::Error, Debug)]
pub enum SdkError {
    #[error("rpc error: {code}: {message}")]
    Rpc {
        code: tonic::Code,
        message: String,
        error_details: Option<Box<ServerErrorDetails>>,
    },
    // ...
}
```

- `ServerErrorDetails` is `Box`ed ‚Äî `clippy::result_large_err` otherwise (the `HashMap<String, String>` context field inflates the variant).
- Decode via `inferadb_ledger_raft::proto::ErrorDetails` + `prost::Message::decode()`.

## Common mistakes

| Mistake | Consequence | Fix |
| --- | --- | --- |
| Forgot `#[snafu(implicit)] location` on a source-bearing variant | Compile succeeds; `.context()` captures location-less trace; debugging is painful | Add the field; every source carries a location |
| Manual construction `MyError::X { source, location: snafu::Location::default() }` | Trace points to the constructor line, not the failing call site | Use `.context(XSnafu)?` |
| Added an error variant but skipped `ErrorCode` mapping | SDK sees `ErrorCode::Unknown`; not retryable; no suggested action | Update `ErrorCode` + the `code()` impl together |
| `anyhow::Result<T>` in a server crate | Foreign error type; strips structure; `snafu-error-reviewer` agent blocks | Convert to snafu |
| `thiserror::Error` in a server crate | Same ‚Äî wrong boundary | Convert to snafu |
| Built `tonic::Status` by hand in a handler | `ErrorDetails` missing; SDK shows bare message | Route through `status_with_correlation` |
